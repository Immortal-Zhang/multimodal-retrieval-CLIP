# 项目报告：基于 OpenCLIP 的双向图文检索原型

## 1. 项目目标

本项目围绕 CIFAR-10 小规模图文对构建一个双向图文检索原型，重点不是训练大模型本身，而是完成“数据准备 → 图像与文本编码 → 双向检索 → Recall@K 评测 → 页面演示”的完整闭环，并以此作为多模态方向的学生项目。

## 2. 数据来源与规模

当前版本基于 CIFAR-10 训练集自动构建图文对数据。默认配置下，每个类别抽取固定数量样本，并为每张图片生成中英双语 caption。

## 3. 方法流程

### 3.1 数据准备

使用 `prepare_cifar10_dataset.py` 下载 CIFAR-10，并生成 `data/captions.csv` 与 `data/images/` 下的图片文件。

### 3.2 向量编码

使用 OpenCLIP 将图片和文本映射到同一向量空间。编码后对向量做 L2 归一化，使点积等价于余弦相似度。

### 3.3 双向检索

- 文本到图像：查询向量与图像向量库做相似度匹配
- 图像到文本：查询向量与文本向量库做相似度匹配

### 3.4 查询扩展

针对中文短查询，使用轻量同义词扩展策略，将多个查询变体分别编码后取平均，再进行检索。

### 3.5 评测

- Text → Image：采用同类别命中作为 Recall@K 标准
- Image → Text：采用 exact match 作为基线评测方式

## 5. 项目价值

本项目的主要价值在于：

- 能完整展示多模态检索的主流程
- 能解释图像和文本如何映射到同一向量空间
- 能实现双向检索与基础评测
- 能展示查询扩展和近似最近邻索引等改进方向
- 能通过页面形式做可视化演示

## 6. 局限与改进方向

可从以下几个方向继续扩展：

1. 增加更真实的图文对数据，而不只限于 CIFAR-10
2. 加入 reranker 或更强的语义后处理
3. 对比不同 OpenCLIP 模型和不同预训练权重
4. 扩展更系统的查询扩展策略
5. 加入更完整的误检案例分析

## 7. 总结

本项目完成了一个小规模双向图文检索原型，能够作为多模态方向的入门项目与简历项目使用。它适合展示“理解检索闭环、会做基础评测、会搭建演示页面”的能力，而不是展示底层预训练或系统优化能力。
